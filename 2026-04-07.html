Agent Authentication & Security New

Runlayer — MCP Security Gateway

Runlayer emerged with an $11M seed round to build a dedicated security gateway for Model Context Protocol (MCP) and multi-agent systems. The company positions MCP as the new attack surface for LLM applications, providing cryptographic agent authentication, privilege delegation, and real-time threat detection. As MCP adoption accelerates in enterprise deployments, a security layer designed specifically for agent-to-resource communication becomes essential infrastructure — especially in regulated environments where non-repudiation and audit trails are mandatory.

Agent Authentication & Security New

Aembit — Workload IAM

Aembit secured an $18M seed round to bring workload identity and access management to autonomous agents. The company's platform extends traditional IAM to handle agent-to-agent credential exchange, dynamic privilege escalation, and continuous compliance verification for agentic systems. This addresses a critical gap: traditional IAM was designed for human users and static services, not for software agents that spawn new identities, rotate credentials mid-execution, and operate across cloud and on-premise systems autonomously. Aembit signals that workload IAM is becoming a foundational infrastructure category.

Hadrius's focus on the most heavily scrutinized regulators (SEC, FINRA) suggests this is the vanguard of compliance-by-design infrastructure for agent-driven finance.

ValidMind is providing continuous AI governance and model monitoring infrastructure for financial services, addressing the SEC's explicit 2026 exam priority around AI capability validation. The company helps firms document, test, and audit AI models used in decision-making — exactly what examiners will demand evidence of. Memory New

Mem0 — Universal Agent Memory

Mem0 raised $24.5M in Series A funding, with AWS as an exclusive early partner, to build enterprise-grade memory infrastructure for autonomous agents. The company's positioning around unified memory across agent instances, tasks, and deployments addresses a critical gap: agents need persistent, contextual memory to operate intelligently over time, but existing solutions (vector databases, context windows) don't provide the abstraction layer required for multi-agent coordination. Mem0's AWS exclusivity signals that cloud providers see agent memory as a core strategic capability.

Memory New

Cognee closed a €7.5M funding round to bring structured, graph-based memory to enterprise agents. Unlike vector-based approaches, Cognee builds semantic knowledge graphs that capture relationships between entities, decisions, and outcomes — enabling agents to reason over historical patterns and avoid repeated mistakes. The company's European backing and focus on knowledge structure suggests deep demand from enterprises managing complex, long-running agentic workflows where simple vector similarity is insufficient for intelligent action.

Payment Rails New

Machine Payments Protocol — Stripe + Tempo Standard

Stripe and Tempo have jointly advanced a draft payment protocol specifically for machine-to-machine (agent-driven) transactions, establishing a standard for how agents authenticate payments, request approval, and execute transfers. The protocol builds on existing payment infrastructure (rails, card networks, bank APIs) but adds agent-specific primitives: cryptographic proof of agent identity, time-bounded execution windows, and automatic reversal on agent failure. This signals that payment infrastructure providers view agent transactions as a distinct category requiring purpose-built standards.

Research

Compliance New

Type-Checked Compliance (arXiv 2604.01483)

A new paper on type-checked compliance systems that enable agents to prove regulatory compliance through formal verification, rather than post-hoc audits. The approach uses type systems (from programming language theory) to encode regulatory rules, allowing agents to compile and execute only rule-compliant actions. This represents a shift from "did the agent comply?" to "the agent cannot execute non-compliant actions."

Safety New

ClawSafety (arXiv 2604.01438)

ClawSafety proposes a safety framework for agentic systems that use function calling and tool use, addressing the risk that agents might misuse available tools. The paper introduces constraint-based tool wrapping and runtime verification to ensure agents only invoke tools appropriately. Critical for any system where agents have access to sensitive APIs or resources.

Authentication New

AIP Agent Identity Protocol (arXiv 2603.24775)

AIP (Agent Identity Protocol) standardizes how agents prove and revoke identity across distributed systems, addressing the challenge of impersonation and credential replay attacks. The protocol builds on established identity standards (OIDC, OAuth) but adds agent-specific semantics for time-limited execution windows and delegated authority chains. Published research on protocol design suggests standardization discussions are accelerating in academic communities.

Security New

SentinelAgent (arXiv 2604.02767)

SentinelAgent presents a framework for detecting adversarial attacks on autonomous agents, including prompt injection, reward hacking, and resource exhaustion attacks. The paper provides detection heuristics and mitigation strategies that can be deployed in production agent systems. The focus on adversarial robustness suggests security research is now following agentic systems into deployment environments.

Memory New

Hierarchical Memory Orchestration (arXiv 2604.01670)

This paper proposes a hierarchical memory architecture for agent systems, organizing memories by temporal scope (working memory, episodic memory, semantic memory) and governance level (local to a task, shared across agents, organization-wide). The framework enables agents to reason about which memories are relevant and how to balance memory consistency across distributed teams of agents. Academic research on memory hierarchy suggests this will become a standard architectural pattern.
